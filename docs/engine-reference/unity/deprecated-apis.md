# Unity — Deprecated / Removed APIs (Don't use X → Use Y)

Target: APIs a coding agent might reach for out of stale (pre-2025) training data that are deprecated, obsolete, or removed by **Unity 6.3 LTS**. Confirmed against official Unity docs/upgrade guides where noted; anything unverified is flagged explicitly.

| Don't use | Use instead | Why / since when | Confidence |
|---|---|---|---|
| `Object.FindObjectOfType<T>()` | `Object.FindFirstObjectByType<T>()` | Deprecated since Unity **2023.1** (predates this project's training-gap window but still commonly generated by LLMs from older habits). Old method did unnecessary InstanceID sorting. | Verified (official docs) |
| `Object.FindObjectsOfType<T>()` | `Object.FindObjectsByType<T>(FindObjectsSortMode.None)` (or `.InstanceID` if you need the old sort order) | Same as above — deprecated since 2023.1. `FindObjectsSortMode.None` is notably faster since it skips sorting. | Verified (official docs) |
| Any `[SerializeField]` on a **property**, method, or type | `[field: SerializeField]` on an auto-property, or plain `[SerializeField]` on a backing **field** only | As of 6.3, misuse is a **compile-time error**, not just a no-op/warning. | Verified (official upgrade guide) |
| `RenderGraphSettings.enableRenderCompatibilityMode = true` (or writing to it at all) | Rewrite custom render passes against the **Render Graph API** (`RecordRenderGraph`, `RenderGraphContext`) | Compatibility Mode deprecated in 6.0, **removed in 6.3**; the property is now read-only and always `false`. | Verified (official upgrade guide) |
| Custom `ScriptableRenderPass.Execute(ScriptableRenderContext, ref RenderingData)` as the primary/only override for new render passes | `ScriptableRenderPass.RecordRenderGraph(RenderGraph, ContextContainer)` | Render Graph is the default path since 6.0; the legacy `Execute` pattern is tied to the now-removed Compatibility Mode. | Verified (official docs), but exact API signature should be double-checked against current URP package docs before writing non-trivial custom passes |
| `AccessibilityNode.selected` | `AccessibilityNode.invoked` | Renamed/deprecated in 6.3. | Verified (official upgrade guide) |
| `PlayerSettings.Android.androidIsGame` | Android **App Category** Player Setting (set to "Game") | Superseded in 6.3 alongside Android 16 large-screen orientation behavior changes. | Verified (official upgrade guide) |
| Round or "legacy" Android launcher icon slots | Adaptive icons only | Deprecated in 6.3. | Verified (official upgrade guide) |
| `UPM_NPM_CACHE_PATH` environment variable (CI/build scripts) | `UPM_CACHE_ROOT` (parent directory, no 1:1 replacement — adjust paths accordingly) | No longer supported as of 6.3; no direct replacement variable. | Verified (official upgrade guide) |
| Experimental lightmapping `AdditionalBakedProbes` API | N/A — removed | Removed in 6.3. | Verified (official upgrade guide) |
| Experimental lightmapping `CustomBake` API | `LightTransport.IProbeIntegrator` | `CustomBake` is now obsolete in 6.3. | Verified (official upgrade guide) |
| `NetworkTransform.Update` override (Netcode for GameObjects) | `NetworkTransform.OnUpdate` (or implement `INetworkUpdateSystem` for authority logic) | No longer overridable as of the 6.3-era NGO version. Only relevant if multiplayer is added later. | Verified (official upgrade guide) |
| Unity Magic Leap XR Plugin (new projects) | N/A — deprecated for new projects, existing projects only | Deprecated in 6.3; irrelevant unless XR is in scope. | Verified (official upgrade guide) |
| Facebook Instant Games platform target | Web platform build target | Deprecated in 6.3, scheduled for removal. | Verified (official upgrade guide) |
| `Window > Search > Index Manager` | `Preferences > Search > Indexing` | Index Manager window removed in 6.3 (Editor UI, not a scripting API, but worth knowing if writing editor tooling/docs). | Verified (official upgrade guide) |

## Things that sound deprecated but are NOT confirmed removed — verify before assuming

- **Legacy `Input` class / Input Manager (`Input.GetKey`, `Input.GetAxis`, etc.)** — still functional via the "Active Input Handling: Both/Old" project setting. Not removed, but the **new Input System package is the recommended default** for new projects since Unity 6.0. Treat legacy `Input.*` calls as "works but not preferred" rather than "deprecated/broken." Do not assume it has been removed.
- **UGUI (`UnityEngine.UI`, Canvas-based UI)** — still fully supported and shipped, **not deprecated**. Unity's own guidance is that UGUI is in "maintenance mode" (receives fewer new features) while UI Toolkit gets active investment, but UGUI is not going away and remains a valid choice, especially for simple HUDs or fast prototypes. See `current-best-practices.md` for nuance.
- **`AssetDatabase`/legacy `AssetBundle` APIs (non-Addressables)** — no explicit removal found in the 6.3 upgrade guide. Addressables is the recommended system for new content-delivery work, but raw `AssetBundle` APIs were not found to be deprecated/removed as of 6.3. If a coding session needs to confirm, check the current Addressables/AssetBundle manual pages directly rather than assuming based on this doc.
- **Specific Input System package version numbers / mobile touch API signatures for 6.3** — not independently verified in this research pass. If touch/gesture input code is being written, pull the current Input System package manual (`docs.unity3d.com` → Packages → Input System, matched to the version resolved in this project's `manifest.json`) before writing code, rather than trusting either training data or this document for exact method signatures.
